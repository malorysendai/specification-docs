# Specification Documents Repository

This repository contains all Product Requirements Documents (PRDs) and technical specifications.

## Structure

- `approved/` - Approved and active PRDs
- `drafts/` - PRDs in development
- `templates/` - Template documents

## Standards

See:
- [PRD_STANDARDS.md](./PRD_STANDARDS.md)
- [TASK_STANDARDS.md](./TASK_STANDARDS.md)

## Process

1. Create PRD in `drafts/`
2. Follow templates and standards
3. Get approvals
4. Move to `approved/`

## Compliance

All PRDs will be automatically checked for compliance with the standards. Non-compliant PRDs will be moved to the `drafts/` folder with a compliance report.
